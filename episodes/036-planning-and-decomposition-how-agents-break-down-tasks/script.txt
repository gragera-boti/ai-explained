Planning and Decomposition — How Agents Break Down Tasks

Imagine you're asked to cook a three-course meal. If you just stood in the kitchen and said "cook dinner," nothing would happen. You'd be overwhelmed by all the possible things to do. So instead, you plan. You decide on the courses, pick recipes for each, list ingredients you need, check what you already have, start the longest-cooking dish first, prep ingredients, cook each course in order, and plate everything at the right time. That's planning and decomposition — breaking a big goal into smaller, manageable steps. And for AI agents, it's one of the hardest and most important capabilities to get right.

At the simplest level, task decomposition is exactly what it sounds like. You give an agent a complex goal — "book a business trip to Chicago" — and the agent needs to figure out the sub-tasks: check the calendar for available dates, search for flights, compare prices, check hotel availability, verify the budget, and finally book everything. The question is how the agent decides what those sub-tasks should be and in what order to do them.

There are three main approaches to planning in AI agents. First, there's implicit planning, where the agent figures out the steps on the fly as it goes. This is what a ReAct agent does — it thinks, acts, observes, and iterates without ever writing down a formal plan. It's flexible and adapts well to unexpected situations, but it can also get lost or stuck in loops because there's no roadmap.

Second, there's explicit planning, where the agent pauses at the start, generates a full plan, and then executes it step by step. This is like writing down your entire recipe before you start cooking. The benefit is that the agent has a bird's-eye view — it can spot dependencies, avoid redundant work, and estimate how long things will take. The downside is that plans can become brittle. If something unexpected happens — a flight gets cancelled, a tool returns an error — the plan needs to be rebuilt from scratch.

The third approach is hierarchical planning, which is the most sophisticated. The agent creates a plan with sub-goals, and each sub-goal can have its own sub-plan. This mirrors how humans actually work. You don't plan every chop of every vegetable — you plan "prep vegetables" as a step, and within that, you decide the details as you go. Hierarchical planning lets agents operate at different levels of abstraction, making the plan both comprehensive and flexible.

One of the most common planning techniques used in AI agents today is Chain of Thought planning. Before taking any action, the model generates a step-by-step reasoning trace. This can be done once at the start or continuously throughout the task. A variant called Plan-and-Solve prompting asks the model to first generate a plan, then execute it. Studies have shown this significantly outperforms asking the model to figure things out as it goes, especially for tasks with many interdependent steps.

Then there's Tree of Thoughts, which takes planning a step further. Instead of generating one linear plan, the model explores multiple possible paths simultaneously. At each decision point, it considers several options, evaluates which ones look most promising, and pursues the best ones. This is computationally expensive — you're effectively running the model multiple times in parallel — but for complex reasoning tasks like solving math problems or planning logistics, it can dramatically improve success rates.

There's also a fascinating technique called ReAct with Reflection, where the agent plans, executes, and then analyzes what went wrong before planning again. After completing a task or hitting a failure, the agent reflects on the experience and generates a "lesson" that informs its next plan. This turns every interaction into a learning opportunity, and over time, the agent gets better at planning tasks it has seen before.

A practical challenge in agent planning is what researchers call the grounding problem. Plans are generated using language, but the real world doesn't always match the model's expectations. An agent might plan to "check email," but if the email tool requires authentication and the agent didn't account for that, the plan fails immediately. Good agent designs handle this by making plans provisional — the agent executes with the understanding that plans can and will need to change.

The state of the art in agent planning is increasingly using something called a planner-verifier loop. One agent generates a plan, while another agent reviews it for errors, missing steps, and potential pitfalls. The verifier sends the plan back with feedback, and the planner iterates until the plan is solid. This mirrors how human teams work — one person drafts, another reviews. It adds latency but dramatically improves reliability.

So whether your agent plans implicitly as it goes, explicitly upfront, or hierarchically with nested sub-goals, the core insight is the same. Planning is what separates a random-action agent that stumbles toward solutions from a deliberate agent that can handle complex, multi-step tasks reliably. The best agent systems combine multiple planning approaches, adapt their planning strategy to the task at hand, and always treat plans as living documents that can change.

That's planning and decomposition. Every big win starts with a small step. See you next time.
